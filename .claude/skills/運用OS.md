# /運用OS - 日次運用OS実行スキル

日次運用OSスクリプト（daily-ops-todo.ts）を実行し、結果を分析してアクションを提案する。

## 手順

1. `npx tsx apps/backend/daily-ops-todo.ts` を実行する（タイムアウト300秒）
2. **実行結果を `apps/backend/daily-ops-result-YYYYMMDD.txt` に保存する**（日付はJST当日）
3. 出力結果を読み取り、以下を整理して報告する:
   - **⑨ ファネルボトルネック**: 各導線の粗利進捗・ボトルネック
   - **⑧ アカウント別実績**: オプト/フロント/個別予約の導線別サマリー
   - **⑪ 仮説検証**: RUNNING中の仮説の進捗
   - **⑩ 効果測定**: 停止CR・配信中CRの評価結果
   - **⑦ 個別予約CPO**: KPI達成/超過のCR一覧
   - **①② 再出稿/横展開候補**: 優先度付きアクションリスト
4. ユーザーが番号を指定したら、該当アクションを実行する（再出稿・横展開等）

## ルールファイル
- `apps/backend/daily-ops-rules.md` にFBベースの思考ルールが蓄積されている
- R001〜R011: 個別予約CPO判断、枯れ検知、フロント売れないCR除外、スプシCV取得、スキルプラスSmart+禁止等

## 注意
- 実行には `.env`（DB接続、Google SA、TikTok APIトークン）が必要
- 実行時間: スプシ読み込みが多いため2〜3分かかる場合がある
- タイムアウトは300秒に設定すること
