# Smart+ BUDGET_MODE_DYNAMIC_DAILY_BUDGET 動的日予算問題

## 問題

Smart+広告グループは`BUDGET_MODE_DYNAMIC_DAILY_BUDGET`（動的日予算）でしか作成できない。
TikTokはこのモードの広告グループに対し、**配信状況に応じて通常adgroup API側の予算を自動拡張する**。

- Smart+ APIで`budget: 3000`と設定しても、通常adgroup APIでは`budget: 300000`になる場合がある
- V2がSmart+ APIで予算をリセットしても、通常API側は自動拡張された値のまま
- TikTokダッシュボードは通常API側の値を表示する → ¥300,000と表示される

## 検証結果（2026-04-09）

- Smart+作成直後: 通常APIも¥3,000（正常）
- Smart+ APIで更新: 通常APIも連動して更新される（正常）
- しかし時間経過で通常API側が¥300,000等に自動拡張される
- `BUDGET_MODE_DAY`への変更は不可（Smart+は`DYNAMIC_DAILY_BUDGET`のみ対応）
- 通常APIでのadgroup更新も`Upgraded Smart Plus ads`エラーで拒否される場合がある

## 対策

### 1. tiktok.service.ts `createSmartPlusAdGroup`
Smart+広告グループ作成直後に通常APIで同じ予算を書き込む（初期値固定）。
`Upgraded Smart+`エラーの場合は許容。

### 2. budget-optimization-v2.service.ts 予算増額・リセット
Smart+ APIだけでなく通常APIにも必ず予算を書き込む。
従来: Smart+ API → 成功 → 終了（通常API側が放置される）
修正後: Smart+ API → 通常API → 両方に書き込み

### 3. redeploy-ad.ts
広告グループ作成直後に通常APIで予算同期。

### 4. cross-deploy.service.ts
`createSmartPlusAdGroup`経由なのでtiktok.service.tsの修正で対応済み。

## 根本原因

`BUDGET_MODE_DYNAMIC_DAILY_BUDGET`はTikTokが「動的に」予算上限を調整する仕様。
Smart+ APIでの予算設定は「基準値」として機能し、TikTokが配信パフォーマンスに応じて
通常API側の予算を引き上げる。これ自体はTikTokの仕様であり回避不可。

対策は「通常API側にも正しい予算を書き込み続ける」ことで、TikTokの自動拡張を上書きする。

## 影響範囲

全Smart+広告グループが影響を受ける。全アカウント共通。
