# スマプラ（Smart+マルチクリエイティブ広告）作成ルール

## 「スマプラ作って」とは

**1つのSmart+広告に複数動画をcreative_listとして格納する**こと。
TikTokが最適な動画を自動選択して配信する。

## 構成

```
1キャンペーン → 1広告グループ → 1広告（creative_listに複数動画）
```

- 個別広告を複数作るのは**スマプラではない**（それは個別出稿）
- UTAGE登録経路は**1つだけ**（LP URLが1つのため）
- ad_text_listも**1つ**

## 作成手順

### 同一アカウント（動画が既にある場合）

1. UTAGE登録経路を**1つだけ**作成（最新CR番号 + 1）
2. 各動画のカバー画像を取得＆アップロード（ユニークファイル名で）
3. キャンペーン作成
4. 広告グループ作成（**ディープファネル + 手動ターゲティング + 除外オーディエンス**）
5. 広告作成（creative_listに全動画、landing_page_url_listは1件、ad_text_listは1件）
6. GETで `deep_funnel_toggle: 'ON'` を**必ず検証**

### 別アカウントに出す場合（横展開スマプラ）

1. 元アカウントから動画をダウンロード（`preview_url`経由）
2. 出稿先アカウントに動画をアップロード（`/v1.3/file/video/ad/upload/`）
   - レスポンスの`data`は**配列**。`data[0].video_id`で取得する
   - `video_signature`はMD5ハッシュ
3. カバー画像取得を待機（動画処理に10〜15秒かかる）
4. 以降は同一アカウントと同じ手順（UTAGE → キャンペーン → 広告グループ → 広告）
5. DB（Advertiserテーブル）から `pixelId`, `identityId`, `identityAuthorizedBcId` を取得して使う
6. 除外オーディエンスはアカウントごとに異なる（下記参照）
7. `deep_funnel_event_source_id`もアカウントの`pixelId`を使う

## API仕様の制約

- `landing_page_url_list`: **最大1件**（複数不可）
- `creative_list`: 複数動画を配列で渡す
- `ad_text_list`: 1件
- カバー画像（image_info）: 各クリエイティブに**必須**。ないとエラー
- カバー画像アップロード: レスポンスの`image_id`フィールドを使う（`web_uri`はundefined）
- ファイル名は毎回ユニークにする（重複でエラー: Duplicate material name）
- 動画アップロードレスポンス: `data`は**配列**。`data[0].video_id`

## ディープファネル最適化（AI導線必須）

**Smart+では`deep_external_action`は効かない。**以下のパラメータを使う:

```typescript
deep_funnel_optimization_status: 'ON',
deep_funnel_optimization_event: 'SHOPPING',
deep_funnel_event_source: 'PIXEL',
deep_funnel_event_source_id: advertiser.pixelId,  // アカウントごとのピクセルID
```

- **create時にしか設定できない**（updateでは反映されない）
- GETレスポンスでは `deep_funnel_toggle: 'ON'` として返る
- 作成後に必ずGETで`deep_funnel_toggle: 'ON'`を検証すること
- SNS/スキルプラス導線には適用しない

## 広告グループ共通設定

| パラメータ | 値 | 備考 |
|-----------|-----|------|
| `targeting_optimization_mode` | `MANUAL` | 年齢指定を反映するために必須 |
| `optimization_event` | `ON_WEB_REGISTER` | 登録最適化 |
| `pixel_id` | DBのAdvertiser.pixelId | アカウントごと |
| `targeting_spec.age_groups` | `['AGE_25_34', 'AGE_35_44', 'AGE_45_54']` | 25-54歳 |
| `placements` | `['PLACEMENT_TIKTOK']` | TikTokのみ |
| `comment_disabled` | `true` | |
| `budget` | `3000`（AI/SNS）/ `5000`（スキルプラス） | |

## アカウント別設定

### ピクセルID・Identity

DBの`Advertiser`テーブルから取得する:
```typescript
const adv = await prisma.advertiser.findUnique({
  where: { tiktokAdvertiserId: advertiserId },
  select: { pixelId: true, identityId: true, identityAuthorizedBcId: true },
});
```

### 除外オーディエンス（excluded_audience_ids）

| アカウント | ID | 用途 |
|-----------|-----|------|
| **全AI共通** | `194977234` | TikTokAIオプトイン（全期間） |
| AI_1 | `194405484` | TikTok用除外オーディエンス |
| AI_2 | `194405486` | TikTok用除外オーディエンス |
| AI_3 | `194405488` | TikTok用除外オーディエンス |
| AI_4 | `194416060` | TikTok用除外オーディエンス |

**加えて、各アカウントで「債務整理」を含むオーディエンスを動的に検索して除外に追加する:**
```typescript
const audResp = await get('/v1.3/dmp/custom_audience/list/', { advertiser_id: advId, page_size: '100' });
for (const a of audResp.data?.list || []) {
  if (a.name?.includes('債務整理')) excludedIds.push(String(a.audience_id));
}
```

## 配信開始タイミング（15時ルール）

- JST 15時より前 → 当日付けで即配信
- JST 15時以降 → 翌日0時から配信（schedule_start_time = 当日15:00 UTC）

## キャンペーン名の命名規則

```
YYMMDD/スマプラ/元CR番号列挙/LP{N}-CR{新番号}
```

例: `260409/スマプラ/CR01131_CR01172_CR01169/LP1-CR01192`

## よくあるエラー

| エラー | 原因 | 対処 |
|--------|------|------|
| `landing_page_url_list: maximum number of items is 1` | LP URLを複数渡した | 1つにする |
| `video material should have 1 image_info as the video cover` | カバー画像が未設定 | 各creativeにimage_info必須 |
| `Duplicate material name` | カバー画像のファイル名重複 | タイムスタンプ+ランダム文字列をファイル名に |
| `Internal error, couldn't copy video data` | 一時的エラー | リトライで解決 |
| `deep_funnel_toggle`がGETで返らない | `deep_external_action`を使った | `deep_funnel_optimization_status: 'ON'`を使う |
| 動画アップロード後`video_id`がundefined | `data`が配列 | `data[0].video_id`で取得 |
