## 1. システム全体像

### このシステムは何をするの？

TikTok広告の運用を**自動化**するシステムです。人間が手作業で行っていた以下の作業を自動で実行します：

```
┌─────────────────────────────────────────────────────────────────────┐
│                    【自動化される作業】                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  1. データ収集    → TikTokから広告データを毎日自動取得              │
│                                                                     │
│  2. 成果分析      → 広告の成果（CPA等）を自動計算                   │
│                                                                     │
│  3. 予算調整      → 成果に基づいて予算を自動増減                    │
│                                                                     │
│  4. 広告停止      → 成果の悪い広告を自動停止                        │
│                                                                     │
│  5. アラート通知  → 異常を検知して管理者に通知                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### システムの基本動作

```
  【外部システム】                【このシステム】              【管理者】
       │                              │                          │
       │    ①データ取得               │                          │
TikTok ├──────────────────────────────┤                          │
  API  │    広告データ                │                          │

       │                              │                          │

Google ├──────────────────────────────┤                          │
Sheets │    CV数・売上データ          │                          │
       │◄─────────────────────────────┤                          │

       │                              │    ③結果確認              │
       │                              ├──────────────────────────┤
       │                              │    ダッシュボード表示     │

### 4.2 予算最適化処理

**実行タイミング**: 手動実行（「予算調整」画面から）

**目的**: 広告の成果に基づいて、予算を増やすか・減らすか・停止するかを判断して実行する

#### フローチャート

```
┌─────────────────────────────────────────────────────────────────────┐
│                      予算最適化処理 開始                             │
│                    (手動実行ボタン押下)                              │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│              対象アカウントの訴求設定（Appeal）を取得                 │
│               目標CPA、許容CPA、目標フロントCPO等                     │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                   配信中の広告一覧を取得                             │
│                    (ステータス = ENABLE)                             │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                      各広告について繰り返し                          │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                   過去7日間のメトリクスを集計                        │
│              支出、インプレッション、クリック、CV数                   │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│               Google SheetsからCV数を取得                            │
│                  (登録経路でフィルタ)                                 │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                      CPA（顧客獲得単価）を計算                        │
│                    CPA = 支出 ÷ CV数                                 │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
          ┌──────────────────┴──────────────────┐
          │           判断ロジック               │
          └──────────────────┬──────────────────┘
                             │
         ┌───────────────────┼───────────────────┐
         │                   │                   │
         ▼                   ▼                   ▼
┌────────────────┐  ┌────────────────┐  ┌────────────────┐
│ インプレッション │  │ CPA > 許容CPA  │  │ CPA < 目標CPA  │
│ < 5,000        │  │                │  │ AND            │
│                │  │ 成果が悪い     │  │ 支出 > 閾値    │
│ データ不足     │  │                │  │ 成果が良い     │
└───────┬────────┘  └───────┬────────┘  └───────┬────────┘
        │                   │                   │
        ▼                   ▼                   ▼
┌────────────────┐  ┌────────────────┐  ┌────────────────┐
│   継続配信     │  │   広告停止     │  │  予算30%増額   │
│   (様子見)     │  │   (PAUSE)      │  │   (INCREASE)   │
└───────┬────────┘  └───────┬────────┘  └───────┬────────┘
        │                   │                   │
        └───────────────────┼───────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────────┐
│               上限日予算チェック（設定がある場合）                    │
│              増額後の予算が上限を超えないよう調整                     │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                   TikTok APIで判断を実行                             │
│            (予算変更または広告ステータス変更)                         │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│                   結果ログを返却・表示                                │
│         (広告名、判断、理由、変更前後の予算等)                       │
└─────────────────────────────────────────────────────────────────────┘
```

#### 判断ロジック表

| 条件 | 判断 | アクション | 理由 |
|------|------|------------|------|
| インプレッション < 5,000 | CONTINUE | 何もしない | データ不足で判断できない |
| CPA > 許容CPA | PAUSE | 広告を停止 | コストが高すぎて不採算 |
| CPA ≤ 目標CPA かつ 支出 > 閾値 | INCREASE_BUDGET | 予算を30%増額 | 好成績なので拡大 |
| それ以外 | CONTINUE | 何もしない | 現状維持で様子見 |

#### 計算式

```
【CPA（顧客獲得単価）の計算】

CPA = 支出金額 ÷ CV数

例：
  支出 = 100,000円
  CV数 = 50件
  CPA  = 100,000 ÷ 50 = 2,000円/件


【判断の閾値（訴求設定で管理）】

目標CPA    = 理想的なCPA（例: 1,500円）
許容CPA    = これを超えたら停止（例: 2,500円）

判断例：
  CPA = 2,000円 の場合
  → 目標(1,500) < CPA(2,000) < 許容(2,500)
  → CONTINUE（継続）

  CPA = 3,000円 の場合
  → CPA(3,000) > 許容(2,500)
  → PAUSE（停止）
```

└────────────────────────────┬────────────────────────────────────────┘
                             │
         ┌───────────NO──────┴───────YES───────┐
         │                                      │
         ▼                                      ▼
┌─────────────────┐            ┌─────────────────────────────────────┐

         │                                        │

                              │

         │                   │                   │

                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐

```

#### 通知の種類

| 通知タイプ | 重要度 | 条件 | 内容 |
|------------|--------|------|------|
| CPA乖離（軽度） | WARNING | 乖離度 20%〜50% | CPAが悪化傾向 |
| CPA乖離（重大） | CRITICAL | 乖離度 50%以上 | CPAが大幅悪化、要対応 |
| レビュー要求 | INFO | 支出10万円ごと | 定期的な確認を推奨 |
| 予算上限到達 | WARNING | 日予算上限に到達 | これ以上増額されない |

## 7. 用語集

### ビジネス用語

| 用語 | 読み方 | 説明 |
|------|--------|------|
| **CPA** | シーピーエー | Cost Per Action（顧客獲得単価）。1件のCV（成約）を得るためにかかった広告費用。`CPA = 支出 ÷ CV数` |
| **CPO** | シーピーオー | Cost Per Order（注文獲得単価）。1件の注文を得るためにかかった広告費用 |
| **CV** | シーブイ | Conversion（コンバージョン）。広告の成果（申込み、購入など） |
| **フロントCV** | - | 初回購入やお試し申込みなど、最初の成約 |
| **インプレッション** | - | 広告が表示された回数 |
| **クリック** | - | 広告がクリックされた回数 |
| **CTR** | シーティーアール | Click Through Rate（クリック率）。`CTR = クリック数 ÷ インプレッション数` |
| **訴求** | そきゅう | 広告で何をアピールするかの設定。目標CPAなどのKPI設定も含む |
| **LP** | エルピー | Landing Page（ランディングページ）。広告クリック後に表示されるページ |
| **Smart+** | スマートプラス | TikTokの自動最適化広告機能 |

### システム用語

| 用語 | 説明 |
|------|------|
| **バックエンド** | サーバー側のプログラム。データベースとの通信、外部API呼び出しなどを担当 |
| **フロントエンド** | ユーザーが見る画面部分。ブラウザで動作するプログラム |
| **API** | Application Programming Interface。システム間でデータをやり取りする仕組み |
| **データベース (DB)** | データを保存・管理する仕組み |
| **バッチ処理** | 定期的に自動実行される処理 |
| **ORM** | Object-Relational Mapping。プログラムからDBを操作しやすくする仕組み（Prisma） |
| **OAuth** | 外部サービス（TikTok）に安全にログインする仕組み |
| **トークン** | 認証情報。TikTok APIを使うための「通行証」 |
| **ページング** | 大量のデータを小分けにして取得する仕組み |
| **JST** | Japan Standard Time（日本標準時）。UTC+9 |
| **UTC** | Coordinated Universal Time（協定世界時）。世界共通の基準時刻 |
